Wybór i opis zbioru danych

Dostępnych jest 10 zbiorów danych. Do opracowania należy wybrać ten, którego numer jest równy ostatniej cyfrze numeru indeksu

Zbiór danych zawiera następujące informacje o studentach:

  1. Numer_indeksu: unikalny numer indeksu studenta (liczba sześciocyfrowa).

  2. Płeć: Płeć studenta, tj. “Kobieta” lub “Mężczyzna”.

  3. Kierunek_studiów: kierunek, na którym studiuje student. Możliwe kierunki to: “Informatyka”, “Ekonomia”, “Psychologia”, “Biologia”.

  4. Wiek: wiek studenta.

  5. Czas_nauki: ilość godzin nauki tygodniowo.

  6. Średnia_ocen: średnia ocen studenta.

Przygotowanie danych

Zadania

  • Załaduj dane (kod, bez kreatora) i przekształć zmienne jakościowe na typ factor.

  • Wyświetl podsumowanie danych. Na podstawie tego określ, które zmienne są jakościowe, a które ilościowe.

Funkcja wyznaczająca kwotę stypendium

Zadania

Napisz funkcję, która na podstawie średniej określi, czy danej osobie należy się stypendium, a jeśli tak, to w jakiej wysokości.

Progi otrzymania stypendium:

  • jeżeli średnia ≥ 4.5, to stypendium wynosi 750 zł,

  • jeżeli średnia ≥ 4.7, to stypendium wynosi 850 zł,

  • jeżeli średnia ≥ 4.9, to stypendium wynosi 950 zł.

Argumentem funkcji powinna być ramka danych.

Funkcja powinna zwrócić listę (lista - jako obiekt w R) złożoną z 3 elementów:

  • liczby przyznanych stypendiów,

  • macierzy/ramki danych z numerami indeksów stypendystów wraz z wyliczonymi kwotami,

  • miesięcznego kosztu stypendium (sumy wszystkich przyznanych stypendiów).

Rozwiązania

stypedia <- function(data) {
  
obliczanie_stypedii <- function(avg) {
    if (avg >= 4.9) {
      return(950)
    } else if (avg >= 4.7) {
      return(850)
    } else if (avg >= 4.5) {
      return(750)
    } else {
      return(0)
    }
  }
  
  data$Kwota_stypendium <- sapply(data$Średnia_ocen, obliczanie_stypedii)
  
  stypendysci <- data[data$Kwota_stypendium > 0, c("Numer_indeksu", "Kwota_stypendium")]

  liczba_stypendiow <- nrow(stypendysci)
  
  koszt <- sum(stypendysci$Kwota_stypendium)
  
  return(list( liczba_stypendiow=liczba_stypendiow, stypendysci=stypendysci, koszt_miesieczny=koszt))
}

Na podstwie wykresu możemy zauważyć, że istnieje zależniść liniowa pomiędzy zmiennymi, ponieważ gdy Student chce mieć lepsze oceny to musi więcej czasu poświęcić na naukę.

Testowanie postaci modelu liniowego : p-wartość wynosi 3.171e-08, więc istnieje statystycznie istotna zależność liniowa między tymi zmiennymi.

Test Shapiro-Wilka test normalnosci :p-wartość jest większa od przyjętego poziomu istotności,więc reszty mają rozkład normalny.

Test Breuscha-Pagana dla homoskedastyczności: p-wartość jest większa niż przyjęty poziom istotności to nie odrzucamy hipotezy zerowej, że wariancja jest jednorodna.

Testu Durbin-Watsona :p-wartość jest większa niż przyjęty poziom istotności więc nie ma statystycznych dowodów na występowanie autokorelacji reszt.

Residuals vs Fitted : Punkty na wykresie są dość losowo rozproszone wokół czerwonej linii. To wskazuje, że model spełnia założenie liniowości.Czerwona linia jest dosyć płaska, że średnia reszt nie zależy od wartości dopasowanych, co jest pozytywne.

Q-Q Residuals : Większość punktów układa się wzdłuż linii prostej y=x, co wskazuje, że reszty mają w przybliżeniu rozkład normalny.

Scale_location : Punkty są stosunkowo równomiernie rozmieszczone wokół czerwonej linii, co sugeruje jednorodność wariancji (homoskedastyczność).

Residuals vs Leverage : Większość punktów znajduje się w granicach przerywanych linii, co sugeruje, że dane są stabilne i nie ma obserwacji wywierających nadmierny wpływ na model.
